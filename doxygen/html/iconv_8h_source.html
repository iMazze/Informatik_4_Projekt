35 /* We would like to #include any system header file which could define
36  iconv_t, 1. in order to eliminate the risk that the user gets compilation
37  errors because some other system header file includes /usr/include/iconv.h
38  which defines iconv_t or declares iconv after this file, 2. when compiling
39  for LIBICONV_PLUG, we need the proper iconv_t type in order to produce
40  binary compatible code.
41  But gcc's #include_next is not portable. Thus, once libiconv's iconv.h
42  has been installed in /usr/local/include, there is no way any more to
43  include the original /usr/include/iconv.h. We simply have to get away
44  without it.
45  Ad 1. The risk that a system header file does
46  #include "iconv.h" or #include_next "iconv.h"
47  is small. They all do #include <iconv.h>.
48  Ad 2. The iconv_t type is a pointer type in all cases I have seen. (It
49  has to be a scalar type because (iconv_t)(-1) is a possible return value
50  from iconv_open().) */
